Scoring Method

This is intentionally simple enough to use weekly without building a whole backend first.

1. Normalize each rank

Every source rank becomes a 0-100 score using a simple formula: 100 - (rank - 1) * 4, capped at 20.

2. Compute hotness

Weighted blend of all four sources. OpenRouter carries the highest weight because it reflects public ecosystem pull most directly.

3. Add capability + business

Capability score reflects task fitness; business score reflects actual value to Crazyrouter growth and product positioning.

4. Output priority

P0 means support and feature aggressively. P1 means support or evaluate. P2 means keep on radar. P3 means no need to highlight.
